Getting Started

What is World Forge?

World Forge is a single HTML file. Open it in any browser, build a world, close the tab. No account, no server, no internet required after the first load. Your world lives in your browser and exports as a JSON file you can back up, move, and reload anywhere.

The app is built on one bet: that the most useful thing you can do for a world isn't write more of it — it's connect what you already have. Places, people, factions, beliefs, conflicts, lore — they all link to each other, and the relationship web fills itself in as you work. Almost every feature exists to help you keep a growing world coherent.

Start with a blank world or one of the ten starter templates. Add entries, cross-reference them, explore the relationship web, run a generator when you need a spark.

A 60-second tour
  1. The sidebar on the left is your map of the app. Sections are grouped (World, People, Lore, Story, Tools).
  2. The Dashboard is your homepage — a live snapshot of your world's current state.
  3. Each entry section (Characters, Regions, Factions, etc.) is a card grid. Click any card to view it in detail mode, or click the ✎ icon to edit directly. Use the + Add button to create a new entry.
  4. Entries connect by name reference — type a faction's name in a character's "Affiliations" field, and the connection appears automatically in the Relationship Web.
  5. Use Settings to customize sections, change theme, manage tags, and export/import your world.
  6. Press Ctrl/Cmd + S any time to save. Export regularly to keep a backup JSON on your device.
Starter templates

The first time you open World Forge with no existing world, a picker appears with four starting points:

  • Empty Slate — completely blank. Best if you want to build from nothing.
  • Generic High Fantasy ("Aldenmere") — a pre-populated world with characters, factions, regions, religions, story threads, codex entries, and tags. All entries cross-reference each other so the relationship web populates immediately. Great for exploring the app or as a starting point to edit from.
  • Low-Magic Grimdark ("The Iron Concord") — political-intrigue setting demonstrating quieter, conflict-driven worldbuilding.
  • Steampunk ("Brassmoor") — industrial-age setting demonstrating non-medieval fantasy.

The picker also has an "import a JSON file" button if you're returning with a previous backup.

Once a world is saved, the picker won't show again. To start fresh, go to Settings → Data → Clear All Data (this cannot be undone — export first!).

Saving, importing, and exporting

Auto-save: Every edit saves immediately to your browser's local storage (IndexedDB with localStorage as fallback). There's also a periodic auto-save every 60 seconds.

Manual save: Click the ✦ Save button in the top-right header, or press Ctrl/Cmd + S.

Export (JSON backup): Click ⬇ Export in the header, or go to Settings → Data & Export and click "Export as JSON File". Downloads a single .json file — your real backup. Keep copies somewhere safe (Dropbox, Google Drive, etc).

Export (Markdown): In Settings → Data, click ⬡ Export Full World as Markdown for a human-readable .md file. Good for sharing or pasting into other tools. Not for re-importing — use JSON for that.

Import: Click ⬆ Import in the header. Warning: importing replaces your current world entirely. If you have unsaved work, export it first.

Why this matters: Browsers can clear local storage in some situations (private/incognito mode, "clear browsing data," extreme storage pressure). The exported JSON file is your real backup.

Sections

Dashboard → Open

The Dashboard is a panel of widgets that summarize your world at a glance. Each widget can be turned on or off individually in Settings → Appearance.

  • ⚡ Story Pulse — counts of active conflicts, open story threads, characters lost, etc.
  • 📖 World at a Glance — your world's name, summary, and a few key stat pills. Click "Edit" to update them.
  • 🩺 Health Checks — flags entries that are missing details, are unusually short, or look stub-like. A nudge to flesh things out.
  • ✦ Random Spark — surfaces an entry you haven't touched in a while, with a prompt to revisit it. Click "↻ Reroll" for a different one.
  • 🔗 Most Connected — lists your most cross-referenced entries (the hubs of your world).
  • ✅ Open Tasks — your active to-do items. Add new ones in Developer Notes.
  • 🆕 Recently Added — the last few entries you created or edited.
Entries (characters, regions, factions, etc.)

Most of World Forge's sections are "entry sections" — collections of items with a consistent set of fields. The eleven built-in entry sections are:

  • Regions & Locations — places: cities, kingdoms, dungeons, wilds.
  • Characters — people, named NPCs, protagonists.
  • Factions & Organizations — groups with goals: kingdoms, guilds, cults, orders.
  • Races & Species — distinct peoples and creatures with culture-bearing potential.
  • Cultures — ways of life, customs, languages, values.
  • Religions & Beliefs — gods, faiths, philosophies.
  • Creatures & Beings — non-character creatures: monsters, animals, spirits.
  • Artifacts & Items — significant objects, named weapons, magic items.
  • Conflicts — wars, feuds, struggles (past, ongoing, brewing).
  • Story Threads — plotlines, hooks, ongoing tensions.
  • Lore Codex — myths, legends, in-world knowledge that may or may not be true.

Common to every entry:

  • Click any card to open it in detail view (clean read mode with clickable references). Click ✎ to edit, ✕ to delete.
  • ⧉ Duplicate — copies the entry and inserts it directly below, with "(Copy)" appended to the name. Great for entries that share a lot of the same fields.
  • ☆ Pin — pins the entry to the top of the card grid so it always stays visible. Click ★ again to unpin. Pinned cards get a gold corner ribbon.
  • Fill bar — the thin line at the bottom of each card is a completion indicator. It fills gold as you add more details. A quick visual cue for stub entries that still need work.
  • Sort select — each section toolbar has a sort dropdown: Name A–Z, Name Z–A, Newest First, Oldest First, or By Type. Pinned entries always float to the top regardless of sort.
  • Search & filters — use the search box to filter by name or text. The × button clears it. Field dropdowns (Status, Type, etc.) are also filters; they highlight gold when active. Click a tag chip on any card to instantly filter the whole section by that tag.
  • Every entry can have an Image, General Notes, and Tags (at the bottom of every form). See Images and Tags.

Genre-aware section names: section titles (Characters, Regions, etc.) automatically rename to fit your world's genre. Set your genre in Settings → General or in the World Overview editor. For a Sci-Fi world, "Characters" becomes "Crew & Contacts," "Regions" becomes "Planets & Sectors," and so on. Fantasy keeps the default names.

Detail view (reading entries)

Clicking any entry card opens a detail view — a clean, read-only display of the entry. This is different from the edit form: it's designed for browsing and linking, not data entry.

What's in the detail view:

  • Image hero (if the entry has one)
  • Section pill, name in big serif type, and a subtitle (role/type/category)
  • All filled-in fields, in a two-column layout. Empty fields are skipped — no clutter.
  • Tag chips, if any are applied.
  • Smart references. Mentions of other entries in any text field become clickable links. So if a character's "Affiliations" says Order of the Silver Bough, you can click straight through to that faction.
  • Mentioned by section at the bottom — every other entry that references this one shows up as a chip. Click to navigate to it.
  • Buttons: Close dismisses the view, ⬡ Copy MD copies the entry as formatted Markdown to your clipboard (useful for pasting into Notion, Obsidian, Discord, etc.), ✎ Edit opens the edit form.

Navigating between entries: reference links and "mentioned by" chips open the next entry in the same detail view — no need to close and re-open. You can navigate your whole world this way.

Keyboard shortcut: Esc closes the detail view.

Timeline & History → Open

The Timeline holds historical events in your world — the spine of your world's story. Events are grouped by Era, sorted chronologically, and displayed with a vertical spine and color-coded dot markers.

Stats bar — the top of the timeline shows a quick summary: total events, number of eras, year span, and legendary event count.

Era sections — events are grouped by their Era field. Click any era header to collapse or expand that era. Each era header shows a count badge, year range, and a + Add button that opens a new event pre-filled with that era.

Expanding events — click any event card to expand it in place and read its full description and linked entity chips. Click again to collapse. Use the ✎ button on hover to open the edit form.

Type colors — each event's left border and timeline dot are color-coded by type: crimson for War/Death/Disaster, teal for Discovery/Birth, purple for Magical/Religious, amber for Political/Plague, gold for everything else.

Filters — the toolbar has three filter dropdowns: Type, Era (populated from your actual events), and Importance (Minor/Normal/Major/Legendary). Use them together to focus on a specific slice of history.

The "Sort Year" field is important. The "Date" field is for display ("Year 1032, 3rd Age") but doesn't sort numerically. Set "Sort Year" to a plain number like 1032 for correct chronological ordering. Events without a Sort Year fall to the bottom.

Linked entities — type a character, faction, or location name in the relevant event fields to create clickable chips. Unmatched names appear dimmed as a broken-link hint.

Use ⇣ Oldest first / ⇡ Newest first to flip the sort direction across all eras.

Relationship Web → Open

An interactive force-directed graph showing how everything in your world connects. Drag nodes to rearrange, scroll to zoom, drag empty space to pan, hover to highlight a node's connections, click a node to open its detail view.

Nine kinds of nodes can appear: Characters, Factions, Regions, Religions, Races, Cultures, Conflicts, Story Threads, and Artifacts. Each is color-coded — see the legend at the bottom of the web.

Lines are inferred from text fields — there's no separate "make a connection" form. Lines appear when:

  • A character's Affiliations mentions a faction; Relationships mentions another character (e.g. Kira — ally, Lord Vael — rival); Current Location mentions a region; Race matches a Race entry; description mentions a culture.
  • A faction's Allies / Enemies mentions another faction; HQ / description / goals mention a region; Leadership mentions a character.
  • A race's Homeland field mentions a region; a culture's Associated Race mentions a race.
  • A conflict's Factions / Characters / Location fields mention those entities.
  • A story thread's Originator / Location / Factions fields mention those entities.
  • An artifact's Owner field mentions a character.
  • A religion's Where Practiced or Holy Sites mention a region.

Line colors: gold = character relationship, crimson = faction/leader, teal = location / ally / practiced, red dashed = enemy, purple = religion, amber = race lineage / homeland, leaf = culture, bright crimson = involved in conflict / story.

Filter dropdown: narrow the graph to a specific node type — Characters Only, Factions Only, Characters & Factions, Races, Cultures, Conflicts, Story Threads, or Artifacts. Useful when the full graph gets dense and you want to focus on one layer.

Zoom controls: + and − buttons in the corner, or scroll with the mouse wheel. ⟲ resets the view.

Tip: names must match exactly (case-insensitive) for a line to draw. If a connection isn't appearing, check spelling.

Maps → Open

Upload any image — a hand-drawn map, an Inkarnate export, a satellite render, a dungeon floor plan — and place pins on it linked to your world entries. You can have as many maps as you like, each with its own pins.

To create a map: click + Add Map and pick an image file. The image is auto-compressed and stored inside your world file. The map opens immediately.

Two toolbar modes:

  • ✋ Pan — drag to move around the map, scroll (or use the zoom controls) to zoom in and out.
  • 📍 Drop Pin — click anywhere on the image to place a pin. A picker appears asking which entry to link it to. You can link pins to any entry type — Regions, Characters, Factions, Quests, Artifacts, or Custom sections — not just regions. Each entry type gets a different pin color.

🏷 Labels toggle: click to show or hide the name label under every pin. Useful when you have many pins close together.

Zoom controls: + and − buttons in the bottom-right corner of the map, plus a % readout. Click ⊙ to reset zoom to 100%.

Pin interactions:

  • Hover a pin to see the linked entry's name.
  • Click a pin (in Pan mode) to open that entry's detail view.
  • Right-click a pin (or long-press on touch) to remove it.
  • Pins linked to deleted entries appear in red — right-click to remove them.
  • Pins linked to entries you've since moved appear in amber.

Rename a map using the text input in the toolbar. Delete it with the ✕ Delete button (this removes the map and all its pins; the linked entries themselves are unaffected).

Lore Codex → Open

The Codex is for in-world knowledge: myths, prophecies, common beliefs, scholarly articles, propaganda, etc. Each entry has a "True or Believed?" toggle so you can record both what people think is true and what's actually true behind the scenes.

Use the "Visibility" field to mark whether something is public knowledge, a rumor, or a secret. Use "Who Believes It" to attribute a belief to a faction or culture.

Developer Notes & Tasks → Open

A scratchpad for the worldbuilder, not the world.

  • To-Dos — small tasks ("flesh out the southern continent," "name the king's third son"). Three priority levels. Type a task and press Enter or click + Add. Open tasks also appear on the Dashboard.
  • Dev Log — entries describing what you worked on. Useful for tracking progress over time.
  • Ideas & Concepts — high-level narrative threads you're tracking.
  • Unanswered Questions — open worldbuilding questions you haven't resolved yet.
  • Future Plans — foreshadowing and seeds you want to develop later.
Generators → Open

Random idea sparks for when you're stuck or just want to browse. Seventeen generators, most with a genre selector so results fit your world. Four have a Save button that creates a fully-populated entry ready for editing.

Use the Genre Filter at the top to show/hide genre-specific generators (e.g. Magic System only for Fantasy, Tech & Gadgets only for Sci-Fi). This filter auto-syncs to the genre you set in Settings — so if your world is Cyberpunk, it pre-selects Cyberpunk for you. Supported filters: Fantasy, Sci-Fi, Modern, Horror, Post-Apocalyptic, Cyberpunk, Steampunk, Western, Historical, Superhero.

Customization

Themes & appearance → Open

In Settings → 🎨 Appearance you can:

  • Pick a theme — 12 options across dark, light, and vivid palettes: Codex (default — warm parchment on dark leather), Verdant (deep forest greens), Gilded (dark amber and gold), Ember (deep crimson and red), Arctic (cool ice blues), Daylight (light parchment), Void (near-black with electric violet), Obsidian (volcanic dark with molten orange), Sage (light botanical sage-green), Dusk (velvet plum with rose-gold), Steel (gunmetal with silver-blue), and Bloom (cream-petal with deep rose).
  • Override the accent color with a custom color picker. Click "Reset" to restore the theme's default.
  • Pick a typeface: Libre (default — classic serif), Cinzel (the original fantasy look), Cormorant, Playfair, Spectral, IM Fell, or Josefin. Non-default fonts load from Google Fonts on first use.
  • Pick a font scale: Small, Normal, Large, or X-Large — scales all text globally via CSS zoom.
  • Toggle individual dashboard widgets on or off.

All appearance settings save to your world file, so they travel with your world on export/import.

Branding (icon & app name) → Open

The very top of Settings → 🎨 Appearance has a "Branding" block where you can change the icon and name shown in the top-left header.

  • Icon — paste any emoji (🐉, ⚔, 🏰, ✨, 🌙, 🔥) or a single character. Replaces the default 🌿.
  • App Name — replaces "World Forge." Also updates the browser tab title.

These save to the world file, so each world can have its own brand. If you blank either field, it falls back to the default.

Useful when you've built a specific named world and want it to feel like its own tool — "Aldenmere" with a dragon icon, "The Iron Concord" with a sword, etc.

Tags → Open

Tags are global labels you define once and apply to entries across any section. Examples: Northern Realm, First Age, Important NPCs, Needs Editing.

To create: go to Settings → 🏷️ Tags, type a name, click + Add Tag. Pick a color from the dropdown. Duplicates (case-insensitive) are blocked automatically.

To apply: open any entry. At the bottom of every entry form is a strip of tag chips — click any chip to toggle it on (filled) or off (outlined). Save the entry.

What you see: applied tags show as chips at the bottom of each entry's card. The Tags settings tab shows usage counts for each tag.

Deleting a tag removes it from every entry that uses it (with a confirmation prompt).

Custom sections → Open

The 11 built-in sections cover most fantasy worlds, but if your world needs something they don't — Vehicles, Spells, Plants, Languages, Trading Routes — you can add a custom section.

To add: Settings → ✦ Customize Sections → + Add Custom Section. Give it a name. A configure dialog opens automatically — set its icon, group, and fields.

Custom sections appear in the sidebar like built-in ones. Their entries support all the same things: tags, images, general notes, search, and they show up in the Dashboard's "Recently Added," "Random Spark," and word counts.

Reordering and grouping: drag any section up or down in the section editor to reorder. Change its "Group" to move it into a different sidebar category (World, People, Lore, Story, Tools).

Deleting: built-in sections can be hidden but not deleted. Custom sections can be deleted (with confirmation — this destroys all their entries).

Custom field types

When you configure a custom section's fields, each field has a Type:

  • Text — single-line input.
  • Long Text — multi-line textarea.
  • Dropdown — a select with options you provide.
  • Yes/No — a boolean toggle.
  • Date — a proper date picker.
  • Number — numeric input. You can specify a unit suffix (e.g. kg, kph, gold pieces).
  • Multi-Tag — type a value, press Enter or comma to confirm a chip; click a chip to remove it. Stored as a comma-separated string for easy export.
  • Image — full image widget with preview, URL input, and file upload (auto-compressed). Same as the built-in Image field.

Built-in sections (Characters, Regions, etc.) have a fixed set of fields, but you can rename their labels — see Renaming & relabeling.

Renaming & relabeling sections and fields

Don't like the default name "Story Threads"? Want "Characters" to be "Heroes & Villains"? Want "Threat Level" renamed to "Power Score"? You can.

Open Settings → ✦ Customize Sections. Each section has:

  • A label field — the section's display name (and the singular form for "+ Add" buttons is auto-derived).
  • A singular field if the auto-derive isn't quite right (e.g. "Cities" vs "City").
  • An icon picker — emoji or any character.
  • A group dropdown — which sidebar category it lives in.
  • A visibility toggle — hide sections you don't use.
  • A configure button (gear icon) — opens the field editor where you can rename individual field labels for built-in sections, or fully redefine fields for custom sections.

Group editor: at the top of the General tab, you can rename, reorder, or add/remove the sidebar groups themselves.

Event Types editor: the timeline's event-type dropdown is editable in Settings → 📜 Event Types.

Features

Global Search

Press / (or click 🔍 Search in the header) to open the Global Search overlay. It searches every entry in every section simultaneously — characters, regions, factions, codex, everything.

  • Results are grouped by section, showing up to 8 matches per section.
  • Click any result to open it in detail view.
  • Press Esc to close without navigating anywhere.
  • Works on names, descriptions, summaries, and most text fields.

For per-section filtering (status, type, tags), use the search box and filter dropdowns within each section instead. Global Search is for when you know roughly what you're looking for but don't remember which section it's in.

Undo

World Forge keeps a 20-step undo history per section. Press Ctrl/Cmd + Z (when not typing in a text field) to undo the last add, edit, or delete.

  • A toast appears at the bottom of the screen after each action, letting you know what was done and that you can undo it.
  • Undo is per-section — undoing in Characters only affects Characters, not Factions, etc.
  • The undo history is in-memory only. Refreshing the page clears it. Export regularly as your real safety net.
Quick Notes (Scratch Pad)

Click 📝 Notes in the header, or press Ctrl/Cmd + Shift + N, to open the Quick Notes scratch pad — a floating panel for jotting down ideas without interrupting your flow.

  • Notes auto-save to your world file after a short pause. They travel with the world on export/import.
  • The panel is draggable — grab the header bar and move it anywhere on screen.
  • Press Esc (when focused inside the pad) or click × to close it.
  • Use it for session notes, half-formed ideas, names to try later, or questions to come back to. When an idea is ready to be an entry, copy-paste it into the proper section.
Markdown export

World Forge can export your lore as Markdown — great for pasting into Notion, Obsidian, a wiki, Discord, or any Markdown-aware tool.

  • Copy one entry as Markdown: open any entry in detail view and click ⬡ Copy MD. The full entry (all fields, tags, notes) is copied to your clipboard as formatted Markdown.
  • Export the whole world as Markdown: go to Settings → Data & Export and click ⬡ Export Full World as Markdown. This downloads a .md file with every section, every entry, all fields.

The JSON export (⬇ Export as JSON File) is still your primary backup format — it's what you import to restore your world. Markdown export is for sharing and reading, not for re-importing.

Genre-aware section labels

World Forge works for any kind of world — not just fantasy. Set your world's genre in Settings → General → Genre / Tone, and the app adapts:

  • Section names in the sidebar and panels rename to fit your genre. Examples: for Sci-Fi, "Characters" becomes "Crew & Contacts," "Regions" becomes "Planets & Sectors," "Magic" becomes "Science & Tech Systems." For Cyberpunk, "Factions" becomes "Corps & Gangs." For Horror, "Bestiary" becomes "Monsters & Horrors."
  • The Generator Genre Filter auto-syncs to your world's genre so genre-specific generators are highlighted automatically.
  • Section subtitles update to match the tone of your genre.

Supported genres include: High Fantasy, Dark Fantasy, Grimdark, Science Fiction, Space Opera, Horror, Cosmic Horror, Cyberpunk, Steampunk, Dieselpunk, Post-Apocalyptic, Superhero, Western, Historical, Modern / Contemporary, Thriller, and more. Fantasy is the default and keeps all original labels.

You can also set genre in the World Overview editor (click your world's name on the Dashboard).

Tips

Adding images to entries

Every entry has an Image field at the bottom of its form. Two ways to use it:

  • Paste a URL — fast, doesn't bloat your world file. Best for images you've hosted somewhere (Imgur, Discord CDN, etc.). Caveat: if the image gets deleted from that host, it disappears from your entry too.
  • Upload a local file — click the 📁 Upload button. The image is stored as base64 inside your world JSON. Best for permanence and fully offline use, but bloats the file.

For uploaded images: anything under ~500KB is fine. The app will warn you if you try to embed a file over 1MB.

Images appear as a thumbnail at the top of the entry's card and float to the right of the entry in the printed PDF.

Linking entries to each other

World Forge doesn't have explicit "select an entry to link to" pickers. Instead, it does name matching: type the name of another entry into the relevant field, and the app finds the link.

Examples:

  • In a character's Affiliations: Order of the Silver Bough, Crown Council.
  • In a character's Relationships: Kira — sister, Lord Vael — rival, Old Master Han — mentor. The text after a dash is just for your benefit; the linker uses the name before it.
  • In a faction's Allies or Enemies: comma-separated faction names.
  • In a religion's Where Practiced: comma-separated region names.
  • In a timeline event's Characters Involved, Factions Involved, or Locations Involved: comma-separated names.

Names match case-insensitively. If a name doesn't match anything (typo, or you haven't created the entry yet), the link still appears in the timeline as a broken chip — a hint to fix it.

Storage & backups

Your world is stored in your browser using IndexedDB (with localStorage as a fallback). This is durable — refreshing or closing the tab won't lose anything — but it's not the same as a real backup.

The Export pill in the header shows when you last exported a backup. It changes color: gray (recent), amber (over two weeks old), red (never). Click it to export immediately.

Storage Diagnostics panel in Settings → Data & Export shows:

  • Your world file size (the JSON dump of everything)
  • How many images you have and how much space they take
  • Last export time
  • Browser storage quota (typically 5GB+, you're nowhere near using it)
  • Per-section entry counts
  • Warnings if images are eating significant space, or if you've never exported

Image compression: when you upload an image, anything wider than 1600px gets resized and re-encoded as JPEG at 85% quality. Tiny images skip this. The toast tells you the size reduction. This keeps your world file lean.

When you might lose your world:

  • Browser data is cleared (manually or by extensions)
  • You're using private/incognito mode (data is wiped on close)
  • You switch browsers or devices and didn't import a backup
  • Browser corruption (rare, but happens)

Best practice: export weekly, keep the JSON file in cloud storage you trust.

Troubleshooting

"My changes aren't saving!"

  • Make sure your browser allows local storage. Private/incognito windows usually clear it on close.
  • Look for a red error toast at the bottom of the screen. World Forge has a global error reporter that surfaces any JavaScript error — if you see one, screenshot it.
  • Try clicking ✦ Save manually (or Ctrl/Cmd + S). If a toast says "Saved ✦", you're good.

"A button does nothing."

  • Hard-refresh your browser (Ctrl/Cmd + Shift + R) — sometimes a cached old version is loaded.
  • Open developer console (F12) and check the Console tab for red error text.

"My world disappeared!"

  • Check whether you're using the same browser profile / device. Local storage is per-browser-per-profile.
  • If you exported a backup JSON file recently, use ⬆ Import to restore it.
  • If not — local storage may have been cleared. Always export periodically.

"I clicked a card and got a read view, where's the edit form?"

  • That's the detail view. Click the Edit button at the bottom-right to open the edit form.
  • For quick-edit (skipping the detail view), click the small ✎ pencil icon on the top-right of the card itself.

"The relationship web is empty."

  • You need at least two entries with a name reference between them. See Linking entries.
